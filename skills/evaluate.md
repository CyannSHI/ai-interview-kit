# AI外呼对话质量评估

## 使用流程

用户触发后，按以下4步执行：

### Step 1：读取数据 + 确认上下文

1. 读取Excel文件，**按表头名称匹配**以下字段：

| 需要的信息 | 匹配表头关键词 | 用途 |
|---|---|---|
| 通话ID | 通话记录id | 标识每通通话 |
| 通话时长 | 通话时长 | 筛选有效样本（≥60s） |
| 对话内容 | 对话内容 | 评估主体 |
| 调研背景 | input_调研背景 / input_研究目标 | 项目上下文 |
| 访谈主题 | input_访谈主题 | 项目上下文 |
| 核心问题 | input_研究目标 | 信息收集核查的依据 |
| 用户背景 | input_经历描述 | 辅助判断 |

如果表头与上述不一致，列出实际表头让用户确认映射关系。

2. 从匹配到的字段中提取**项目上下文**（调研背景、访谈主题、核心问题清单），向用户确认。
3. 筛选有效通话（时长≥60s且对话内容非空），将所有有效通话暂存为中间数据（JSON格式）。
4. 在对话中告知用户：共X条记录，Y通有效，即将开始评估。

### Step 2：分批评估

**每批最多5通对话**，按以下流程处理：

1. 读取当前批次的对话文本（仅当前批次，不要一次性读取全部）。
2. 逐通评估（评估框架见下文），为每通对话产出结构化结果。
3. 将本批次结果追加写入临时结果数据。
4. 在对话中简要报告进度：`已完成 X/Y 通`。
5. 继续下一批次，直到全部完成。

**批次结果数据结构**（每通对话一条记录）：
```json
{
  "call_id": "通话ID",
  "duration": 96,
  "has_problem": true,
  "bad_cases": [
    {"code": "B2", "name": "上下文遗忘", "desc": "问题描述", "evidence": "原文引用"}
  ],
  "info_collection": {
    "total": 5,
    "collected": 2,
    "details": [
      {"question": "Q1简述", "status": "✅/⚠️/❌", "note": "说明"}
    ]
  },
  "exit_reason": "用户挂断",
  "remark": "备注（如无效样本）"
}
```

### Step 3：生成结果Excel

全部评估完成后，生成Excel文件，保存在源数据同目录下，文件名：`评估结果_[访谈主题]_[日期].xlsx`。

Excel包含4个Sheet：

**Sheet 1「走查总表」** — 每通通话一行

| 列 | 内容 | 说明 |
|---|---|---|
| # | 序号 | |
| 通话ID | 完整ID | |
| 时长(s) | 秒数 | |
| 有无问题 | ❌有问题 / ✅正常 | |
| Bad Case | 编码+类别名 | 如"B2上下文遗忘, B9流程异常"；无问题填"—" |
| 信息收集 | X/Y | ✅状态的数量/核心问题总数 |
| Q1~Qn | 各问题状态 | ✅/⚠️/❌ ，列头用问题简称 |
| 退出原因 | | |
| 备注 | | 无效样本等特殊标记 |

**Sheet 2「Bad Case明细」** — 每个Bad Case实例一行

| 列 | 内容 |
|---|---|
| 通话ID | |
| 时长(s) | |
| 编码 | B1~B10 |
| 类别 | 封闭诱导/上下文遗忘/… |
| 问题描述 | 一句话概述发生了什么 |
| 证据原文 | 引用对话中的关键片段 |

**Sheet 3「汇总统计」** — 聚合数据，纵向排列

写入以下内容（每块之间空一行）：
- 基础指标：走查通话数、问题通话数/占比、正常通话数/占比
- Bad Case分布表：编码、类别、出现次数、占比、涉及通话ID
- 信息收集概况表：各核心问题的✅/⚠️/❌数量和收集率
- 退出原因分布表
- Top问题及改进建议（3~5条，每条含问题描述+建议）

**Sheet 4「归因分析与行动项」** — 将评估发现转化为可执行的下一步

需要读取用户的Agent Prompt（询问用户提供），对照评估结果进行归因。

写入以下内容：

1. **归因总览表**：每个Bad Case类型一行，标注频次、归因层（Prompt层/模型平台层/ASR层/随机偏差）、是否需要行动
   - 筛选标准：出现≥2次且有相同pattern的才算系统性问题，低频随机case标记为"不改"
2. **Prompt层问题**（可直接修改提示词解决）：每个问题列出现象、Prompt中的根因（引用具体段落）、建议改法、涉及通话
3. **模型/平台层问题**（需反馈技术侧）：每个问题列出现象、根因、行动项、证据原文
4. **随机偏差**（不需要行动）：列出Prompt中已有对应规则的低频case，说明为什么不改
5. **待确认项**：如信息真伪需与产品核实等
6. **行动项清单**：按P0/P1/P2优先级排列，每条含：行动项、负责方、具体操作、预期效果

归因判断原则：
- 如果Prompt中已有对应规则但仍出现 → 归因为模型随机偏差（不改Prompt）
- 如果Prompt中缺少对应约束 → 归因为Prompt层（需要改）
- 如果问题与语音识别、消息时序相关 → 归因为模型/平台层（反馈技术）
- 低频（≤1次）且无固定pattern → 视为随机噪声，不纳入行动项

### Step 3.5：询问Agent Prompt

生成Sheet 1-3后，**询问用户是否提供Agent Prompt文件**用于归因分析。

- 如果用户提供：读取Prompt，对照评估结果生成Sheet 4归因分析
- 如果用户不提供：跳过Sheet 4，仅输出Sheet 1-3

### Step 4：对话中输出摘要

生成Excel后，在对话中**仅输出**：

```
评估完成，结果已保存：[文件路径]

快速摘要：
- 走查 X 通，其中 Y 通存在问题（Z%）
- Top Bad Case：[B_编码类别名]（N次）、[B_编码类别名]（N次）
- 信息收集完整率最低：[问题名]（X%）
- 建议重点关注：[一句话]
```

不要在对话中输出完整的表格或详情，全部进Excel文件。

## 评估框架

### 对话文本格式

对话为 `用户:xxx` / `客服:xxx` 交替的文本，包含特殊标记：
- `【用户挂机】`：用户主动挂断
- `【机器挂机】`：AI主动结束
- `【静音】`：AI执行静音等待
- `no_response`：用户无语音输入（可能在思考，不等于拒绝）

### 一、Bad Case检测

逐项检查以下10类问题。**存在才标记，不强行找问题。每个标记必须引用对话原文作为证据。**

| 编码 | 类别 | 判断标准 |
|---|---|---|
| B1 | 封闭诱导 | AI给出选项或引导性表述："是不是因为XX""比如A还是B""对吗" |
| B2 | 上下文遗忘 | AI忘记用户前面说过的信息，重复或矛盾提问 |
| B3 | 话题偏移 | AI在非核心话题上追问超过2轮，偏离主线 |
| B4 | 机械追问 | AI未理解抽象概念/心理阈值，做字面化机械追问 |
| B5 | 过度追问 | 用户已充分回答或明确拒绝，AI仍不断追问 |
| B6 | 静默误判 | 用户短暂沉默在思考，AI误判为不愿交流，过早结束 |
| B7 | 情感失察 | 用户已不耐烦/困惑/抵触，AI未调整策略 |
| B8 | 信息编造 | AI提及不存在的产品信息、价格、功能 |
| B9 | 流程异常 | 跳过关键阶段、重复阶段、无法正常结束、未识别非目标用户 |
| B10 | 噪声干扰 | 明显的ASR识别错误、语音重叠导致答非所问 |

**Bad Case在任何输出中都必须同时写编码+类别名**，如"B2上下文遗忘"，不能只写编码。

### 二、信息收集核查

对照核心问题清单，逐条核查：
- 该问题是否在对话中被问到
- 用户是否给出了有效回答（区分"回答了"vs"浅层敷衍"vs"未回答"）
- AI是否进行了追问，追问后是否获得了更多信息

用三个状态标记：
- ✅ **已收集**：用户给出了实质性回答
- ⚠️ **浅层**：用户回答了但很笼统（如仅一个词），AI未有效深挖
- ❌ **未收集**：未问到 / 问了但用户未回答

### 三、退出原因

结合对话末尾标记判断：
- **正常完结**：AI输出总结/感谢语后 `【机器挂机】`
- **AI主动结束**：AI判断意愿不高，跳到结束语
- **用户主动挂断**：出现 `【用户挂机】`
- **超时终止**：达到轮次/时间上限
- **静默退出**：连续无响应后系统结束

## 评估原则

- **只基于对话文本中的事实判断**，不推测文本之外的内容
- **引用原文作为证据**，每个Bad Case必须附带具体对话片段
- **结合用户背景**：如用户背景信息显示特定情况，AI应据此调整提问
- **正确理解特殊标记**：`no_response` 可能是思考中；`【静音】` 是AI主动等待行为
- **没有问题就不标记**，不强行找Bad Case
- **信息收集核查要对照核心问题清单**，不凭印象判断
